Communication between Delta Plc And Python

Import Packages

Python provides package called pymodbus in which there are many modules to communicate with PLC
depending on support provided by PLC.
To download packages from pycharm
i. Open pycharm->File->Settings->Project Interpreter.
ii.Click on + sign at the right side or press alt+insert.

iii. Search the package i.e pymodbus and download it
Or you can also install packages directly from the command prompt
pip install pymodbusTCP/ pip3 install pymodbus
This supports both TCP?IP and ASCII communication.
Alternative for pymodbus is pymodbusTCP which can be used for only TCP/IP connections

Modbus TCP/IP Using pymodbus

To communicate using modbus TCP/IP, import ModbusTcpClient
from pymodbus.client.sync import ModbusTcpClient
client = ModbusTcpClient(host='localhost', port=5020)
connection = client.connect()
This will complete your python connection with PLC.

Reading PLC REGISTERS/INPUTS

For reading plc registers we use
reg = client.read_holding_registers(address=0,count= 1, unit=1)
regs1=reg.registers[0]
1.Enter Modbus adress you want to read for eg:(If you want to read a T0 timer in Delta PLC then put the modbus
adress which is 401537 and if this doesn't work the just use 41537 or 01537 since 4 acts
as a offset and pymodbus may not support 6 digit modbus address)
2.Enter count of registers you want to read.
3.Insert the unit ID of PLC which can be found out on a PLC simulators for delta PLC it is WPLSOFTS
Using This User can now read the data present in Modbus Address which user Entered.Just click on
Read Registers and data will be displayed.
Follow similar procedure for reading discrete inputs

Writing PLC REGISTERS/INPUTS

client.write_register(address=0,value=0,unit=1)
Address and unit will remain same
value is the one which you will write to specified PLC tag.


Modbus ASCII communication using pymodbus

client = Modbusserialclient(method='rtu', port=502, baudrate=9600, timeout=3, parity=N, stopbits=1, bytesize=8)
connection = client.connect()
1.Method=rtu or ascii

2.port=Enter port number of PLC

In my case it is COM4. You can check it using device manager

3.baudrate=9600

4.parity=N

This will make connection with PLC
Reading and writing commands remains same.

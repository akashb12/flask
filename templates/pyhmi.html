PYHMI BY NAKSH AUTOMATION

1.Pyhmi is written in python code,It is especially build for PLC's(Programmable Logic Controller) which has modbus TCP/IP or modbus ASCII support.

2.It is free of cost and compact Application

3.It can read PLC registers also Reset registers,at one time can read 2 registers and display their results.

4.It can Read Discrete Inputs and has functionality of providing different colours eg.(If input is high then green colour and if input is low then red colour) and can read 4 discrete inputs in one time

5.It has built in support for SQL Server Database and has graph plotting function which can take any 2 inputs from database


Two buttons for reading registers and 2 buttons for reading discrete inputs

Connect Your PLC with Pyhmi

So as you can see there are two types of communications which can be used to connect PLC with pyhmi
1.Modbus TCP/IP
2.Modbus ASCII

Modbus TCP/IP

So here user needs to provide
1. IPaddress of PLC
2.Port number of PLC
Then click on connect button and your connection will be established between pyhmi and your PLC

Modbus ASCII

1.Method=rtu

2.port=Enter port number of PLC

In my case it is COM4. You can check it using device manager

3.baudrate=9600

4.parity=N

Enter this and click on connect button

Reading Registers And Reset

After successful connection we can now read PLC registers
1.Enter Modbus adress you want to read for eg:(If you want to read a T0 timer in Delta PLC then put
the modbus address which is 401537 and if this doesn't work the just use 41537 or 01537 since 4 acts
as a offset and pymodbus may not support 6 digit modbus address)

2.Insert the unit ID of PLC which can be found out on a PLC simulators for delta PLC it is WPLSOFTS

Using This User can now read the data present in Modbus Address which user Entered.
Just click on Read Registers and data will be displayed

DATABASE CONNECTION

1. Driver = ODBC Driver 11 for SQL Server
It may not be same for everyone so to check drivers you can run this command in python
for driver in pyodbc.drivers(): print(driver)

And before this user needs to create a connection string in ODBC server.
While establishing connection string you need to specify Server name And the database name
Another method to check the driver is open ODBC and go in connection pooling

2. Server = LAPTOP-CNL636HF\SQLEXPRESS
User can check their server name by entering into SQL server database

3. Database Name = Enter the name of the database which you created

4. Tabel Name = Enter the name of the table which you created

5. Coulmn Name=Enter column names in which user wants to enter data.
Initially user can store datetime in one column and register value in another column.
So create 1st column of type datetime and 2nd column of type int.
If user wants to store anything else iplace of datetime and int can contact us at nakshautomation07@gmail.com

6. Trusted Connection = yes

Once user Enters all this details then user can click on READ REGSTER inorder to insert data to SQL server

Plotting Graph

Now after inserting data into the database user can retrieve the data and show it on a graph

1.Enter query=select datetime,count from pytb2

Here datetime and count are column names of database and table name is pytb2.

2.X-Label=Enter name you want to display as X-Label

3.Y-Label=Enter name you want to display as Y-Label

Now User can click on generate graph to view the graph.

Reading Discrete Inputs

Reading Discrete inputs is very simple

Just Enter Modbus Address of discrete input you want to read and enter unit number of PLC and you are all ready to read it.

If input is high then it will turn green

If input is low then it will turn Red

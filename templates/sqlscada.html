Interfacing Intouch SCADA with SQL SERVER

Once user reads PLC tags like counter or timer value then it can be
easily stored into SQL SERVER database.First user needs to create connection string using ODBC.

ODBC connection string



ODBC stands for data source administrator, user first needs to go to
1.First open SQL server database and click on connect button and create a database for storing PLC tags.
2.If you are reading counter value or timer value then they are of integer type so if you want to store
it to database then give proper data types for columns.For storing counter it should be INT type.

While connecting with database if user gets this error then don't worry.

Just go to services(you can search your pc for finding it) scroll down till you find sql server and
check its status if its not showing running then just right click on sql server and click start.

3. NOW set system DSN. For that search for odbc data sources 32 bit in your PC.
Click on Add button

4. Select driver for which you want to set ODBC connection.
User can select SQLSERVER

3. Give name to your ODBC connection
5. You can add description as you want
6. Now add server name which you will get once you open SQL SERVER

Copy the server name and paste it in ODBC

7. Select windows authentication or sql server authentication depending
on what was selected while connecting sql server

8. If here you have selected windows authentication then select same
for ODBC connection and before this please connect with your database.
Create tables in database and keep everything ready and then set ODBC connection else ODBC configuration will
throw an error as connection failed

9. Now that connection is established user can provide database name for which connection is needed.
After that just click on next until you see finish button and that's all for setting ODBC connection.

Using ODBC connection string in SCADA



10. So ODBC is established next step is to provide this ODBC connection string to INTOUCH SCADA.
11. Now comes very important step that is BIND LIST.Bind list is used to link PLC tags which are present
in SCADA and Database columns where you want to store this tags. 12.In Intouch SCADA you will find bind list in tools section.

13. Open bind list and click on new bindlist.
11. Give name to bindlist
12. Now you need to link PLC tags with DAtabase Columns.
13. In tag.fieldname put the same tag name you created and which you want to insert into database.
14. In Column name insert the database column name where you want to store that plc tag.
15. In my case i have created a tag name called bagcount and also a database column name called bagcount.
You can use different names also.

16. After completing bind list let's move ahead to calling ODBC connection string to scada
17. For calling connection string you need to write a simple script

18. ResultCode=SQLConnect( ConnectionId,"DSN=scadazuariodbc;
This is to connect with ODBC string.For this provide the same DSN name which we created before while setting ODBC string.
19. APP=Microsoft Windowns Operating System;WSID=(local);
DATABASE=scada;Trusted_connection=Yes");
You can put your operating system and provide the database name in which you want to store data.
20. ResultCode=SQLInsert( ConnectionId, "scadatb", "scadazuaribl" );
This is for inserting data into database for that give table name for accessing your columns and give bindlist name.
21. Finally end connection with disconnect command.
22. Now script is complete you can press on okay.
Sometimes it will ask you to define the terms result code and connection ID so just define it with same datatype.
23. Memory integer or memory discrete are for simulation purpose they work without connecting with plc.

24. You can add this script to a button which is when pressed will insert everything you put in a bind list to database.
For that you need to select action and write script inside it.
Or you can use data change script which will execute this script on every datachange.
For eg: In my case i am inserting data for everyminute for that go to scripts and select data change
script and select tagname.field as minute for that double click on tagname.field and window will pop.

Retrieve Database values and display in SCADA



25. It is very simple to do.
Just use same script and replace insert with select.

26. ResultCode=SQLFirst( ConnectionId );
This is for retrieve content from 1st row
27. Now to display those values back into SCADA just provide same plc tags to # so that they can properly retrieve the data.

28. For count i have given same BAGCOUNT tag which i had used while inserting it into database.
Now the # will get replaced by retrieved data from database.

Drink Water Reminder

Main aim of this project is to make you understand how to convert any python file to run in background or auto schedule your python script.

To explain you all in a better way I am taking example of a Drink water notification which will remind you to drink water after every one hour.

Writing a code


You need to install plyer library . This library is used to give prompt or notification.

We want our code to notify us after every 1hour. so you can create a while loop which will run after 1 hour, and for doing this you need to import module called time.
Title gives you the title for your notification and you can give any message whatever you wish to.
You can also give the app icon, make sure icon is a file with an extension of .ico otherwise it will show you an error.
Timeout is the time for which you want to display the notification.
And sleeptime is 1hr.

import time from plyer import notification if __name__=="__main__": while True: notification.notify( title="Please Drink Water", message = " total daily water intake: 3.7 liters (15 cups) for <br> the average adult male and 2.7 liters (11 cups) for the average adult female.", app_icon="C:\study material\python\water\mmm.ico", timeout=5 ) time.sleep(60*60)

Now since the code is ready lets learn how to auto-schedule this.
Open the task schedular using search feature in windows. and then go to create task.





Give a name to your task as "drink water reminder" or
anything which relates yourapplication and go to actions to create a new actions.



Under program/script we need to mention the path where the python executable is located.
There are many way to find this, one way is to open the command prompt .
comamnds
>python
>import os
>import sys
>os.path.dirname(sys.executable)
this will give you the path but this is not in windows friendly way.
To make it in a windows friendly format just remove all the
adition / from the path and then apend it with python.exe.
eg:SC:\Users\Nidhi\AppData\Local\Programs\Python\Python38\python.exe


Then you need to give an argument, argument is nothing but the path of the file which you want to run.


If incase your application does not work then cover the argument in double inverted commas.
Then go to trigger, trigger is nothing but when you want to schedule your application. Since we already mentioned in the code we need not do this step.
